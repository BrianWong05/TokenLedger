# Issue tracker: GitHub

Issues and PRDs for this repository live in GitHub Issues at
`BrianWong05/TokenLedger`. Use the `gh` CLI from inside the repository for all
operations so it infers the remote automatically.

## Conventions

- Create an issue with `gh issue create`.
- Read an issue and its discussion with `gh issue view <number> --comments`.
- List work with `gh issue list`, including labels when filtering by workflow
  state.
- Update labels or content with `gh issue edit`.
- Add discussion with `gh issue comment`.
- Close completed or rejected work with `gh issue close`.

## Pull requests as a triage surface

Pull requests are not a feature-request or triage surface for this repository.

## Skill instructions

When a skill says to publish to the issue tracker, create a GitHub Issue. When
a skill says to fetch the relevant ticket, read the GitHub Issue and its
comments.
